Hindsight 0.7.1 is a fast follow-up to 0.7.0 focused on **reliability** — tougher consolidation, safer oversized retains, more correct recall recency, and steadier Codex OAuth embeddings — plus a few useful tuning knobs (LLM reasoning effort, per-provider reranker timeouts) and a bit more i18n polish. We recommend all users on 0.7.0 upgrade.

<!-- truncate -->

- [**Tougher Consolidation**](#tougher-consolidation): Indefinite retry with backoff, dedup-by-bank, and priority-based scheduling so the worker keeps making progress under pressure.
- [**Safer Oversized Retains**](#safer-oversized-retains): Oversized items now stay in one async child to eliminate a foreign-key race.
- [**Correct Recall Recency**](#correct-recall-recency): Recency scoring is anchored to the query timestamp instead of server clock skew.
- [**Codex OAuth Embeddings**](#codex-oauth-embeddings): Token refresh and follow-up fixes for stable long-running embedding workloads.
- [**New Tuning Knobs**](#new-tuning-knobs): Configure LLM reasoning effort and per-provider reranker HTTP timeouts from env.
- [**More Chinese Locales**](#more-chinese-locales): Additional zh variants and a polish pass on the existing translation.

## Tougher Consolidation

The consolidation worker gets two related fixes that together keep it healthy under load:

- **Indefinite retry with backoff.** Previously, a transient failure during consolidation could cause the worker to give up on the bank until the next trigger. 0.7.1 retries consolidation indefinitely with exponential backoff and a per-bank deduplication guard so the same bank can't be picked up twice concurrently and waste cycles.
- **Priority-based bank scheduling.** When multiple banks have pending consolidation, the worker now pulls them in priority order. Higher-priority banks get their consolidation latency back under control in mixed-workload deployments instead of waiting behind large, low-priority banks.

## Safer Oversized Retains

When a retain item was too large to fit in a single LLM call, 0.7.0 split it across multiple async children. That split could occasionally trip a foreign-key race against the parent retain row. 0.7.1 keeps oversized items in a single async child — the LLM-side chunking still happens, but the database-side parent/child relationship stays one-to-one, removing the FK race entirely.

## Correct Recall Recency

The recency component of recall scoring was using the server's current clock as the reference point, which produced subtle drift when callers passed an explicit `query_timestamp` (for example, replaying historical events). 0.7.1 anchors recency to the query timestamp instead, so recall on historical queries returns the same results regardless of when you re-run them. Thanks to [@Sanderhoff-alt](https://github.com/Sanderhoff-alt) for catching this.

## Codex OAuth Embeddings

The Codex OAuth embeddings provider added in 0.7.0 gets two follow-ups in 0.7.1:

- A proper **`CodexAuthManager` with token refresh**, so long-running embedding workloads don't fail when the OAuth access token expires.
- Cleanup of the refresh path to handle a couple of edge cases caught after the initial ship.

If you're using `HINDSIGHT_API_EMBEDDINGS_PROVIDER=openai` with Codex OAuth credentials, this upgrade is worth picking up. Thanks to [@DK09876](https://github.com/DK09876).

## New Tuning Knobs

Two new environment variables let you tune model behaviour without code changes:

- **`HINDSIGHT_API_LLM_REASONING_EFFORT`** — sets the reasoning effort for reasoning models (e.g. `low`, `medium`, `high`). Useful for trading latency against quality on extraction and reflect calls. Thanks to [@s9rkn](https://github.com/s9rkn).
- **Per-provider reranker HTTP timeouts** — separate timeout env vars per reranker provider so you can give slower providers more headroom without globally relaxing the timeout for everyone.

## More Chinese Locales

Building on the 8-language Control Plane shipped in 0.7.0:

- Added additional Chinese locale variants. Thanks to [@MapleEve](https://github.com/MapleEve).
- A polish pass on the existing `zh` translation for more natural phrasing.

## Other Notable Changes

- **`graph_maintenance` filter** in the Control Plane operations dropdown so you can filter for graph-repair runs alongside other operation types.
- **OpenClaw**: flushes un-retained turns on `session_end`, no longer silently skips dispatch on synthetic-main + static-banking setups, and labels "Current time" as UTC in injected memory context.
- **Docs**: a docker-compose example for running Hindsight against a local llama.cpp sidecar, Codex Docker recipe for the Claude Code integration, and BM25 backends table now lists PGroonga and the pg_search tokenizer option.
