Hindsight 0.7.0 is a major release focused on **multilingual search**, **horizontally scaled deployments**, and **graph maintenance correctness**. It adds three new BM25 backends (ParadeDB pg_search for Citus, PGroonga for CJK/polyglot, and a configurable native language), a new ZeroEntropy embeddings provider, Codex OAuth embeddings, a fully internationalized Control Plane in eight locales, and a reactive link-recompute pipeline that fixes outgoing-link staleness after deletes. Observation consolidation quality also gets a substantial prompt rewrite, plus a targeted-by-scope variant.

## ParadeDB pg_search for Citus

`HINDSIGHT_API_TEXT_SEARCH_EXTENSION` gains a fourth option: `pg_search`. ParadeDB's pg_search is the only true-BM25 backend that works on a Citus distributed Postgres cluster, which unblocks horizontally scaled deployments that previously had to fall back to the `native` tsvector backend.

The retrieval arm builds the `@@@` predicate via `paradedb.boolean(should => ARRAY[paradedb.match('text', ...), ...])` so multi-field coverage (text + context + text_signals) is preserved without query-string interpolation. A docker-compose recipe ships under `docker/docker-compose/pg_search/` based on the official `paradedb/paradedb:latest-pg17` image.

A new `HINDSIGHT_API_PG_SEARCH_TOKENIZER` knob also lets you pick the tokenizer (`default`, `whitespace`, `raw`, etc.) when the default doesn't fit your corpus.

## PGroonga + Configurable BM25 Language

Two new knobs and an opt-in backend make non-English banks — especially CJK — work out of the box:

- **`HINDSIGHT_API_BM25_LANGUAGE`** drives the PostgreSQL text search dictionary used by the native tsvector backend (default: `english`). Validated as a PG identifier so it can be safely embedded in `to_tsvector('<lang>', ...)`.
- **`HINDSIGHT_API_RETAIN_OUTPUT_LANGUAGE`** forces the fact extractor to emit facts in a specified language regardless of the source language. Independent from the BM25 language so users can mix indexing/extraction languages deliberately.
- **`pgroonga`** is a new option for `HINDSIGHT_API_TEXT_SEARCH_EXTENSION`. It uses TokenBigram + NormalizerNFKC150, so a single polyglot index handles English, CJK, etc. simultaneously. Ships with its own docker-compose recipe.

To support per-deployment languages, the `GENERATED ALWAYS` expression on `memory_units.search_vector` is dropped via a new Alembic migration, and the column is populated at INSERT time using the configured language.

## Async Link Recompute

When a memory_unit is deleted (via `delete_document`, `delete_memory`, or document re-ingest), the FK cascade removes its incoming temporal/semantic links. Other units that had this unit in their top-K neighbours therefore lose links and stay permanently under-capped — retain only generates links for newly inserted units, never re-evaluates surviving ones.

0.7.0 adds a reactive top-up:

- Inside the delete transaction, `from_unit_ids` that pointed at the doomed units are captured and written to a new `link_recompute_queue` table.
- After commit, a new `link_recompute` task type is submitted, deduplicated per bank.
- The worker drains the queue in batches of 50; for each victim, it counts current outgoing temporal/semantic links and, if below the cap (20 temporal, 50 semantic), re-runs the same probes retain uses and inserts the missing links.

This restores graph-expansion recall correctness after deletes without any change to the hot retain path.

## Control Plane in 8 Languages

The Control Plane UI is now fully internationalized via `next-intl`. Pages move under a `[locale]` segment with locale-prefixed routing (the default English has no prefix), and there's a Globe language switcher in the header. Locales shipped:

- English (`en`) — default, no prefix
- Spanish (`es`), French (`fr`), German (`de`), Portuguese (`pt`)
- Japanese (`ja`), Korean (`ko`), Chinese (`zh`)

The middleware combines i18n routing with the existing auth middleware. Its matcher uses an explicit file-extension allowlist so bank IDs that contain dots (e.g. `SX.Products.GovComply.Build`) still get the correct locale rewrite.

A vitest parity test and a static finder (`scripts/find-untranslated.ts`, exposed as `npm run i18n:check`) walk the TSX AST to flag hardcoded user-facing strings — both wired into CI so future drift fails the build.

## Targeted Consolidation by Scope

The `POST /v1/default/banks/{bank_id}/consolidate` endpoint now accepts an `observation_scopes` parameter. Pass it to re-consolidate only memories whose tags match one of the supplied scope combinations — useful when a tag-policy change invalidates a subset of observations and you don't want to rebuild the entire bank.

A companion flag `enable_auto_consolidation` (configurable per bank) lets you disable the automatic post-retain consolidation entirely, so deployments that prefer to consolidate on a schedule or on-demand can opt out of the default behaviour.

## Consolidation Quality Rewrite

The observation consolidation prompt is rewritten with explicit markdown structure (`## MISSION`, `## PROCESSING RULES`, `## INPUT`, `## DECISION GUIDE`, `## OUTPUT FORMAT`) and a new rule 1 — **PREFER UPDATE OVER CREATE** — that addresses the root cause of duplicate sibling observations (issue #1566).

Other consolidation-side changes:

- **Mission decoupled from behaviour.** The default mission now describes *what* to track; the processing rules describe *how* to consolidate. A mission-priority note tells the LLM the per-bank mission overrides the rules on conflict, so per-bank `observations_mission` cascades cleanly.
- **AT MOST ONE UPDATE PER `observation_id`** + a defensive `_dedupe_updates` guard. The LLM occasionally emitted multiple updates for the same observation in one batch; without dedup the later write silently overwrote the earlier.
- **Two worked examples in the prompt** (merging recurring claim → UPDATE only; state change + unrelated CREATE) replace the previous single create-heavy example.

The reflect-side temporal reasoning surfaced a few related issues that also got fixed in the same pass.

## Clear Mental Model

A new `POST /mental-models/{id}/clear` endpoint resets a mental model's content to empty, so the next refresh performs a full re-synthesis regardless of trigger mode. This is useful for periodic compaction of delta-mode models that accumulate drift over many incremental refreshes.

The Control Plane surfaces this in the mental-models detail view, and SDK examples (Python, Node.js, CLI, Go) are included in the docs.

## New Embedding Options

- **ZeroEntropy** is a new embeddings provider. Set `HINDSIGHT_API_EMBEDDINGS_PROVIDER=zeroentropy` and a model.
- **Codex OAuth embeddings**: a new `openai-codex` embeddings provider that uses the existing Codex OAuth token (no separate API key). OpenAI output dimension overrides (`HINDSIGHT_API_EMBEDDINGS_OPENAI_DIMENSIONS`) are also documented for the 384-dimension configuration path.

## Other Notable Changes

- **Alibaba Qwen3 reranker support** in the local cross-encoder path.
- **Right Agent integration** — a new entry in the integrations catalog.
- **OpenCode Go LLM provider** for the `opencode-go` runtime.
- **Ollama Cloud provider** with corrected native-API authentication for cloud endpoints.
- **API access logging env var** to enable Uvicorn access logs without code changes.
- **Per-operation LLM concurrency caps** to keep parallel workloads from overloading a provider.
- **Pre-normalized reranker scores** are now detected and switched to rank-based normalization automatically.
- **Stop sending `temperature`** to Anthropic — the latest models reject it.
- **Webhook batch deliveries** are deduplicated to avoid duplicate retain notifications.
- **Oversized single items in batch retain** are split before the call, so very large payloads no longer fail the whole batch.
- **Disabled reflect tools** are hidden from the agent's system prompt, not just gated at invocation.
- **Invalid embedding dimensions** no longer trigger retries — the response is rejected immediately.
- **User-defined label entities** are no longer subject to fuzzy resolution, so user-defined identities stay stable.
- **Mental model history** is now capped to prevent JSONB overflow, and full refreshes correctly rebase pending delta baselines.
- **Gzip middleware** is enabled so compressed graph responses stay parseable.
- **Control Plane session cookies** are now verified by signature, not by presence — a meaningful auth hardening.
- **Windows embedded UI startup** resolves the absolute `npx` path before spawning, fixing a long-standing launch failure.
- **Tag groups and triggers** are preserved correctly when updating tags.
